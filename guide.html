A 전체 구조

먼저 읽고 내 위치를 파악합니다.

3단 구조

  • 메인홈 = "공공의주방 홈". 전 영역의 최상위 입구. (파일명 index.html)
  • 테마홈 = 영역별 홈. 7대 영역.
  • 도구 = 각 테마홈 안의 카드 (예: 케이터링 → 견적산출기·메뉴선택기).
  • 이동 흐름: 메인홈 → 테마홈 → 도구.

7대 영역 + 파일명 (영어 고정 · 한글이면 404)

영역파일명
메인홈index.html
업무일지worklog.html
재무·세무finance.html
쿠킹박스cookingbox.html
클래스class.html
케이터링catering.html
마케팅marketing.html

용어 구분

  • 홈 = 도구로 가는 입구(이동). "런처"라고도 함.
  • 대시보드 = 데이터 현황판(숫자). 홈과 별개.

B 데이터 흐름 대원칙

⭐ 가장 중요 · 반드시 먼저 이해할 것

채팅창(작업 대화창)이 최신 원본의 기준점이다. github은 보기·이동용 서버일 뿐, 데이터 저장소가 아니다. (github에서 최신본을 가져온다고 생각하지 말 것)

데이터는 두 곳에서 바뀝니다:

  • ① 채팅창에서 수정
  • ② 테마홈 화면에서 직접 입력·수정

②의 경우, 화면에서 파일을 다운로드해 사용자가 채팅창에 전달합니다. → 이렇게 채팅창을 항상 최신으로 동기화합니다. 따라서 테마홈은 화면의 데이터를 html·엑셀로 다운로드하는 기능이 필수입니다.

무엇을 채팅창에 줄지 (데이터가 어디 담겼느냐에 따라)

  • 데이터가 엑셀 마스터에 따로 있는 홈 → 데이터 바뀌면 엑셀을 준다
  • 데이터가 html 안에 담기는 홈(예: 케이터링) → 데이터 바뀌면 html을 준다
  • 화면·기능(디자인)을 바꿨을 땐 → html을 준다
  • 헷갈리면 둘 다 주면 가장 안전

C 상단 헤더 (위→아래 순서 고정)

(1줄) 왼쪽: 알약형 버튼 [← 공공의주방 홈] (href="index.html") 오른쪽: "ARCHIVEAT · ○○" (영문 영역명, 작은 회색 라벨) (2줄) 작은 라벨: "테마 홈 · ○○" (3줄) 제목: "○○ 홈" (Hahmlet 명조, 크게) (4줄) 날짜·버전: "2026.06.26 · v○" (DM Mono, 회색, 오늘 자동) (5줄) 설명: 아래 D 규칙대로 한 줄
  • 메인홈 가기 버튼은 알약형으로 왼쪽 맨 위 고정.
  • 우상단은 ARCHIVEAT · ○○(영문) 라벨로 통일. (네모박스 X)

D 5줄 설명 정의

  • 형식: "○○부터 ○○까지 관리합니다" — 이 영역이 무엇을 하는 곳인지 한 줄.
  • 예) 케이터링: "행사 등록부터 견적·메뉴·정산까지 관리합니다."
  • 예) 재무·세무: "매출·매입과 분기 신고자료, 결산을 관리합니다."

데이터 출처·신뢰성("재무제표 기준 검증" 등)이나 조작 안내는 여기 넣지 않습니다. → 출처는 카드 안에, 운영 원칙은 맨 아래 ※ 안내(F)로 분리.

E 원본 다운로드 구역 "📦 원본 파일"

목적: 테마홈에 문제가 생겼을 때 원본으로 기준을 다시 잡고, 화면에서 바꾼 데이터를 채팅창에 전달(동기화)하기 위함.

버튼 2개 (둘 다 화면에서 직접 다운로드)

  • 📄 이 화면 HTML 받기 — 현재 화면(입력 데이터 포함)을 통째로 다운로드. 현재 document 전체를 Blob으로 만들어 받는 자기 자신 저장 방식. 파일명 ○○.html
  • 📊 엑셀 마스터 받기 — 현재 화면의 데이터를 .xlsx로 다운로드. 입력형 홈은 즉석 변환, 마스터 있는 홈은 그 마스터를 받기. (github에서 끌어오는 게 아님)

받은 파일은 사용자가 드롭박스에 보관 + 채팅창에 전달해 최신 동기화합니다.

★★ 화면에서 직접 입력하는 홈(localStorage)의 필수 주의 ★★

localStorage = 사용자가 화면에서 입력한 데이터를 브라우저에 저장하는 방식 (케이터링 등).

다운로드 버튼은 반드시 localStorage의 입력 데이터까지 파일에 담아 내보내야 합니다.

❌ 흔한 실수: 버튼이 localStorage를 빠뜨리고 빈 껍데기만 다운로드. (실제로 이 문제로 빈 파일이 받아진 적 있음 → 반드시 방지)

✅ 올바른 동작: "기존 최신본 + 화면에서 새로 입력한 데이터"가 합쳐진 완전한 파일이 다운로드되어야 함.

F 맨 아래 ※ 운영 안내 (모든 테마홈 공통)

※ 홈은 보기·이동만 합니다.
  · 데이터를 고칠 땐 → 마스터(엑셀/대화창)에서
  · 볼 땐 → 이 홈에서
  · 모든 파일은 같은 폴더(github)에 두어야 링크가 작동합니다.

H 기술 원칙 (필수)

  • 단일 HTML 파일. 외부 의존 최소화 (폰트 못 불러와도 시스템 글꼴로 작동).
  • 파일명 ○○.html 고정 (영어). 버전은 푸터에만.
  • 맨 위 복귀 링크: <a href="index.html">← 공공의주방 홈</a>
  • 모든 파일 같은 폴더, 링크는 상대경로만.

localStorage 사용 원칙

  • 원칙적으로 사용하지 않는다 (데이터는 파일·마스터에 둠).
  • 단, 케이터링처럼 화면에서 직접 입력·수정하는 홈은 예외로 허용. → 이 경우 E의 "다운로드 시 localStorage 데이터 포함" 규칙을 반드시 지킨다.

I 충돌 시 · 적용 전 확인

이 지침을 적용하기 전에, 각 테마홈의 기존 구조·기능과 맞지 않는 부분이 있는지 먼저 점검한다.

충돌하거나 이 홈에 맞지 않는 부분이 있으면 → 기계적으로 적용하지 말고, 먼저 사용자에게 알리고 대화한 뒤 작업한다.

단, 아래 필수는 예외 없이 맞춘다: 영어 파일명 · 메인홈 복귀 링크 · 헤더 순서(C의 1~5줄).
